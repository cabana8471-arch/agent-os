# Issue ID Categories

Use unique identifiers: `[CATEGORY]-[NUMBER]` (e.g., SEC-001, BUG-015)

| Category | Code | Description |
|----------|------|-------------|
| Security | SEC | Security vulnerabilities, auth issues |
| Bug | BUG | Functional bugs, logic errors |
| Performance | PERF | Performance issues, bottlenecks |
| Code Quality | QUAL | Code smells, maintainability |
| Documentation | DOC | Missing or incorrect documentation |
| Test | TEST | Missing tests, test failures |
| Architecture | ARCH | Architectural issues, design flaws |
| Dependency | DEP | Dependency issues, version conflicts |
| Feature | FEAT | Feature proposals and opportunities |
| Duplicate | DUP | Duplicate functionality detected |
| Gap | GAP | Missing or incomplete features |

Numbers: 3-digit zero-padded (001-999), sequential per category.
